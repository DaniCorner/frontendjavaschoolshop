Project Overview

My project is about creating a web page that acts like an online movie store. It allowed me to list and categorize all the movies I've watched in my life. The movies are stored and organized in a MySQL database.

For this project, I used MySQL Workbench for managing the database, Spring Boot for the backend, and Angular for the frontend.

The target audience for this project is movie lovers who want a platform to showcase and manage their movie preferences. Users can browse the movie collection, search for titles, and get more details about each film. They can also, as Admins, add, edit, and delete movies from the database.

My main goal is to meet all the requirements specified by the Java School. By completing these objectives, I want to demonstrate my skills in full-stack development, including frontend, backend, and database management.

I think I've met the Java School's requirements delivering an enjoyable user experience for cinephiles and movie enthusiasts.

Database Integration


I integrated the MySQL database into my project using Spring Data JPA, which simplifies the mapping between entities and the database.

Some of the entities included are Product, ProductCategory, Order, OrderItems, Customer, and User. User entities have assigned Roles.

Relationships between entities were made, such as the one-to-many relationship between ProductCategory and Product, using foreign keys.

Using Spring Data JPA, I defined the entity classes with annotations like @Entity, @Table, @OneToMany, @ManyToOne, etc.

Technologies & Frameworks


The frontend components, such as templates, stylesheets, and TypeScript files, are organized following Angular's component-based architecture.

On the backend, Spring Boot provides the server-side logic and API endpoints. MySQL Workbench is the database management system used. Entities representing movies are mapped to database tables using Spring Data JPA.

Unit tests are implemented using Mockito. Postman is used for testing and validating the API endpoints during development. IntelliJ and Visual Studio Code are used as IDEs.

UI Design


For the User Interface of my project, I designed it to be user-friendly and visually appealing. I incorporated Bootstrap and also I made use of various Angular Material components.

I implemented form controls with validation messages to guide users and prevent errors. The application responds promptly to user actions, providing feedback and loading indicators.

DAO Layer


To interact with the database, I implemented the Data Access Object. They can encapsulating the logic for CRUD operations (Create, Read, Update, Delete). JPA provides a set of default CRUD methods, written in SQL.

Exception Handling


In my project, I used try-catch blocks in appropriate places within the code. I also made use of custom exception classes to encapsulate specific types of errors and provide more detailed information about the exception.
